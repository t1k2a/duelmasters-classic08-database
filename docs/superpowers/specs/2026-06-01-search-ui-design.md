# デュエマ クラシック08 カード検索UI 設計書

**作成日**: 2026-06-01  
**ステータス**: 承認済み

---

## 概要

既存の Hono 検索 API (`/api/cards`) を活かしつつ、Next.js でカード検索フロントエンドを構築する。PC ではサイドバー固定＋リアルタイム更新、スマホではインラインフィルター。

---

## アーキテクチャ

### 構成

```
既存 Hono API サーバー (port 3000)  ← 変更なし
    ↑ fetch (NEXT_PUBLIC_API_URL)
Next.js App (port 3001)
```

### ディレクトリ構造

```
app-ui/                          ← Next.js プロジェクトルート（リポジトリ内）
  app/
    layout.tsx                   ← グローバルレイアウト・メタ
    page.tsx                     ← 検索トップ（/ ）
    cards/
      [cardId]/
        page.tsx                 ← カード詳細ページ
  components/
    SearchSidebar.tsx            ← PC用サイドバーフィルター
    MobileFilters.tsx            ← スマホ用インラインフィルター
    CardList.tsx                 ← 検索結果リスト
    CardListItem.tsx             ← カード1枚の表示コンポーネント
    CivilizationTag.tsx          ← 文明カラータグ（再利用）
  lib/
    api.ts                       ← Hono API クライアント（fetch ラッパー）
    types.ts                     ← フロント用型定義（CardResult等）
  .env.local                     ← NEXT_PUBLIC_API_URL=http://localhost:3000
```

### 技術選定

| 項目 | 採用技術 | 理由 |
|---|---|---|
| フレームワーク | Next.js 15 App Router | ユーザー既存スキル・SSR対応 |
| スタイリング | Tailwind CSS v4 | レスポンシブが宣言的に書ける |
| 状態管理 | React `useState` + URL search params | 軽量・ブラウザバック対応 |
| データ取得 | `fetch`（クライアントサイド） | リアルタイムフィルタリング |

#### フィルター仕様

| フィルター | PC | スマホ | API パラメータ |
|---|---|---|---|
| テキスト検索 | テキスト入力 | テキスト入力 | `name` AND `text` 両方を同一キーワードで検索 |
| 文明 | チェックボックス | カラータグ（タップ） | `civilization` |
| コスト範囲 | 数値 2 input | セレクトボックス | `cost_min` / `cost_max` |
| パワー範囲 | 数値 2 input | （スマホは省略） | `power_min` / `power_max` |
| カードの種類 | セレクト | セレクト | `cardType` |
| 種族 | テキスト入力 | （スマホは省略） | `race` |

**リアルタイム更新**: フィルター変更から **300ms debounce** 後に API 呼び出し。  
**URL 同期**: フィルター状態を URL search params に反映（`?civilization=火&cost_max=5` 形式）。ブラウザバック・ URL 共有に対応。

#### 結果リスト（CardListItem）の表示項目

- カード名（太字）
- カードの種類・文明・コスト / パワー
- 種族
- セットコード・カード番号・レアリティ

#### ページネーション

- 1ページ 20 件
- 前 / 次 のシンプルなページング

### 2. カード詳細（`/cards/[id]`）

`id` は DB の `cards.id`（整数）。CardListItem からリンクを生成する。  
Hono 側に `GET /api/cards/:id` エンドポイントを1本追加して対応。

```
┌────────────────────────────────────┐
│ ← 検索に戻る                        │
│                                    │
│  ボルメテウス・ホワイト・ドラゴン    │
│                                    │
│  種類    クリーチャー               │
│  文明    🔴 火                      │
│  コスト  7                         │
│  パワー  7,000                      │
│  種族    アーマード・ドラゴン        │
│                                    │
│  ── 能力テキスト ───────────────── │
│  （DB から取得した実際のテキスト）   │
│                                    │
│  ── 収録セット ─────────────────── │
│  DM-06  S8/S10  SR                 │
│                                    │
│  ── フレーバーテキスト ──────────── │
│  （あれば表示）                     │
│  イラスト: Shishizaru               │
└────────────────────────────────────┘
```

「← 検索に戻る」は `router.back()` でフィルター状態を保持したまま戻れる。

## データフロー

```
ユーザー操作（フィルター変更）
    ↓ setState
React state 更新
    ↓ useEffect + 300ms debounce
GET /api/cards?civilization=火&cost_max=7&...
    ↓ JSON
CardList 再描画
```

カード詳細ページ:
```
/cards/42 アクセス（42 = cards.id）
    ↓ Server Component fetch
GET /api/cards/42  ← Hono に新規追加するエンドポイント
    ↓ JSON（CardResult 形式）
CardDetail 描画
```

## 受け入れ条件

1. `npm run dev`（Next.js）起動後、`http://localhost:3001` で検索できる
2. 文明・コスト・種類・テキストでフィルタリングできる（リアルタイム更新）
3. カード名をクリックで `/cards/[cardId]` に遷移し詳細が見れる
4. スマホ幅（375px）でフィルターが使える
5. URL にフィルター状態が反映され、リロードで状態が復元される
