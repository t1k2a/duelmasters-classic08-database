# カード詳細レイアウト改善 デザイン仕様

**日付:** 2026-06-01  
**対象ファイル:** `docs/design-preview.html`  
**スコープ:** レイアウト変更 + カード画像表示（DB/API 変更なし）

---

## 概要

現在の「左サイドバー＋右メインエリア（検索バー全幅 + リスト + 詳細）」構成を、3カラム均等幅構成に変更し、カード詳細パネルを大型化してカード画像を表示する。

---

## レイアウト

### デスクトップ (md 以上)

```
[左: 検索＋フィルター w-56] | [中央: カードリスト w-56] | [右: カード詳細 flex-1]
```

| カラム | 幅 | 内容 |
|---|---|---|
| 左 | `w-56` (224px) 固定 | テキスト検索バー（上部）+ 既存フィルター（文明・コスト・パワー・カード種類・種族） |
| 中央 | `w-56` (224px) 固定 | カードリスト（カード名・文明バッジ・コスト／パワー・レアリティ） |
| 右 | `flex-1` | カード詳細（カード画像 → 文明バッジ → スタッツ表 → 能力テキスト → 収録情報） |

### モバイル (md 未満)

現状のレイアウト（テキスト検索バー全幅 + 文明タグ + カードリスト）を維持。カードタップ時の底部スライドアップモーダルにもカード画像を追加する。

## カード画像

### 画像 URL

```
https://dm.takaratomy.co.jp/wp-content/card/cardimage/{cardId}.jpg
```

### cardId の構築ルール

モックデータの `set_code` + `card_number` から構築する。

| set_code | card_number | cardId |
|---|---|---|
| `DM-01` | `1/110` | `dm01-001` |
| `DM-01` | `85/110` | `dm01-085` |
| `DM-06` | `S8/S10` | `dm06-s08` |
| `DM-10` | `9/110` | `dm10-009` |
| `DM-30` | `1/55` | `dm30-001` |

**変換ロジック:**
1. `set_code` を小文字化し `-` を除去: `DM-06` → `dm06`
2. `card_number` の `/` 前の部分を取得: `8/110` → `8`、`S8/S10` → `S8`
3. 先頭が数字のみの場合は3桁ゼロパディング: `8` → `008`
4. 先頭が `S` の場合は `s` + 2桁ゼロパディング: `S8` → `s08`
5. 結合: `dm06` + `-` + `008` = `dm06-008`

### エラーハンドリング

`onerror` で `this.style.display='none'` を設定し、画像が存在しない場合は非表示。

### モックデータへの追加

CARDS 配列の各エントリに `card_id` フィールドを追加する（上記ルールで事前計算済みの値を埋め込む）。

## 実装変更点

### 1. HTML 構造の変更

**変更前:**
```
<aside w-56>フィルター</aside>
<main flex-1>
  <div>検索バー（全幅）</div>
  <div>
    <div flex-1>カードリスト</div>
    <div w-80>カード詳細</div>
  </div>
</main>
```

**変更後:**
```
<aside w-56>検索バー + フィルター</aside>
<div w-56>カードリスト</div>
<div flex-1>カード詳細</div>
```

### 2. `detailHTML(c)` 関数の変更

カード画像を先頭に追加:
```html
<div class="flex justify-center mb-4">
  <img src="https://dm.takaratomy.co.jp/wp-content/card/cardimage/{c.card_id}.jpg"
       alt="{c.name}"
       onerror="this.style.display='none'"
       class="rounded-lg shadow-md max-w-[200px]">
</div>
```

### 3. CARDS モックデータへの `card_id` 追加

各カードに `card_id` フィールドを追記する。

### 4. スマホモーダルへの画像追加

`mobileDetailBody` にも同じ画像要素を先頭に追加する。
